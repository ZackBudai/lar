# Datasets

This directory contains datasets organized by source for benchmarking.

## Dataset Sources

### `tptp_syn/` - Syntactic TPTP (without equality)
FOF (First-Order Logic) problems in TPTP format that don't use equality operators.
These are standard logic programming examples from textbooks and literature.

**Files:**
- `ancestor_recursive.p` - Classic recursive ancestor relationship
- `family_transitive.p` - Transitive family relationships
- `function_parent.p` - Functions in first-order logic
- `independent_predicates.p` - Independent predicates (counterexample)
- `knights_knaves.p` - Knights and knaves logic puzzle
- `likes_exists.p` - Existential quantification example
- `modus_ponens.p` - Modus ponens reasoning
- `non_entailed_simple.p` - Non-entailed conjunction
- `socrates_mortal.p` - Classic Socrates syllogism chain
- `tweety_penguin.p` - Default logic / negation as failure example

### `ai_generated/` - AI-Generated Datasets
Datasets generated by AI systems. Add your generated problems here in either format:
- TPTP `.p` files (FOF format)
- JSONL format (one JSON problem per line)

## Running Benchmarks

### Run benchmarks on all sources
```bash
python scripts/run_benchmarks.py
```

### Run benchmarks on specific source(s)
```bash
python scripts/run_benchmarks.py --sources tptp_syn
python scripts/run_benchmarks.py --sources ai_generated
python scripts/run_benchmarks.py --sources tptp_syn ai_generated
```

### Adjust timeout per case
```bash
python scripts/run_benchmarks.py --timeout 10.0
```

### Results
Benchmark results are saved to:
- `results/results_tptp_syn.json` - Results for SYN dataset
- `results/results_ai_generated.json` - Results for AI-generated dataset

### Generate Comparison Figures
After running benchmarks:
```bash
python scripts/generate_comparison_figures.py
```

This generates:
- `figures/source_runtime_comparison.png` - Total runtime across sources
- `figures/source_clauses_comparison.png` - Clause generation comparison
- `figures/speedup_comparison.png` - Speedup ratios by source

## Adding New Datasets

1. **TPTP Format (.p files):**
   Place `.p` files in the appropriate source directory. The script automatically discovers all `.p` files.
   
   Example structure:
   ```
   fof(ax1, axiom, parent(john,mary)).
   fof(goal, conjecture, ancestor(john,mary)).
   ```

2. **JSONL Format (.jsonl files):**
   One JSON problem per line. Each line should contain a complete problem definition.
   
   See existing JSONL datasets for format details.
